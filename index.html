3개의 프로젝트

기계학습을 활용한 유물공간 예측관리시스템(팀 프로젝트 3인)

Contents

Why

유물수명에 가장 많은 영향을 끼치는 요인이 온도, 습도인데, 유물이나 목재, 종이 등을 보관 및 관리하는 공간에서 단순하게 온도, 습도를 테이블형태로만 가져오는 형식이나 수기로 작성하는 형식은 불편하고 비효율적이라고 생각이 들었습니다.
따라서 유물 공간을 관리하는 사람의 입장에서 더 편리하고 효율적인 시스템이 존재한다면 효율적인 관리와 위험 대비, 비용 절감이라는 기대효과를 가져오기 위해 프로젝트를 진행하게 되었습니다.

What

1) 10분 간격 온습도 데이터 저장 - Api를 이용하여 온습도 데이터를 10분 간격으로 저장하고 웹을 업데이트 (자동화)
2) 온습도 시각화 기능 - 관리자가 직관적으로 온습도 데이터의 해석을 가능하게 하는 기능 (직관적인 해석)
3) 온습도 예측 기능- 과거에 데이터를 가지고 미래의 온습도를 예측한 결과를 제시하는 기능 (위험 대비)
4) 위험 알림 기능- 온도나 습도가 안전범위를 벗어났으면 알람을 주는 기능 (효율적인 온습도 관리)

Conception

Used Skills

● HTML, CSS, JS, BootStrap
● Flask, Mongo DB, Aws EC2
● Api, Plotly, TensorFlow
● heyhome Smart Sensor, Hub

My Role

● python Flask를 이용하여 웹페이지 기능구현
● Tensorflow의 LSTM모델을 이용한 온습도 예측시스템
● MongoDB를 이용하여 Database 구축
● AWS EC2의 ubuntu로 서버 활성화

Result


Review:

LSTM 모델을 가지고 온습도를 예측하는 시스템을 개발할 때 공공데이터포털에 과거 기상정보 데이터, 스마트 센서로 가져온 온습도 데이터를 가지고 학습을 진행하였는데 처음에는 성능이 평균 약 85.7%로 좋지 않았던 점이었고, 성능을 높이기 위해 데이터 전처리 및 상관관계를 계산하여 강한 상관관계를 가진 속성만 학습을 진행하는 방법과 정확한 비교를 위해 minmax 스케일러를 사용하여 데이터의 정보를 변형시키지 않고 값의 범위를 같게 변환하는 방법으로 성능을 개선하여 온도는 약 평균 95%, 습도는 약 평균 90%로 개선하여 문제를 해결하게 되었습니다.

Contents

Why

청주 육거리 종합시장은 규모가 크고 점포들이 모여있기 때문에, 밀집도가 높고 화재 피해가 큰 시장이라고 합니다.
이러한 육거리 종합시장을 대상으로 애플리케이션을 통해 화재 위치를 정확히 파악하고 화재 현장 주변의 구조물(아케이트, 소화전 등), 도로 정보를 확인하여 화재 피해를 최소화하고 화재진압 소요 시간을 단축하기 위해서 프로젝트를 진행하게 되었습니다.

What

1) 화재진압에 필요한 데이터 제공

2) 건물 검색기능
3) 유지보수를 위해 건물 좌표, 아케이드 옥상 위치, 진입로 위치 를 추가, 삭제, 수정 할 수 있는 프로그램 제작

데이터 편집 프로그램의

충북대 음식점 추천 사이트(개인 프로젝트)

Contents

Why

충북대 학생으로서 무엇을 먹을지 고민하는 사람들이 있다는 것을 깨닫고 음식의 종류(한식, 중식, 일식, 분식), 위치(정문, 중문, 서문)뿐만 아니라 자신의 의견도 표현할 수 있는 웹 사이트 를 만들어 보자는 생각으로 개발하게 되었습니다.
이러한 웹 사이트가 있다면, 사용자는 위치와 음식의 종류에 따라 음식점을 추천받아 고민하는 시간을 줄이고
평점과 댓글을 보며 다른 사용자의 음식점에 대한 의견을 보고 음식점을 결정할 수 있고 음식점 업주 입장에서는 음식점 등록신청을 통해 가게 홍보와 평점과 댓글을 보고 부족한 점을 보완할 수 있다고 생각하였습니다.

What

1) 위치별로 음식점 추천 - 충북대학교 정문, 중문, 서문별로 구분하여 사용자가 원하는 위치에 음식점을 추천하는 기능
2) 종류별로 음식점 추천 - 한식, 중식, 양식, 분식별로 구분하여 사용자가 원하는 음식의 종류로 음식점을 추천하는 기능
3) 평점별로 음식점 추천 - 사용자가 요청한 평점대의 음식점을 추천하는 기능
4) 평점, 댓글 기능 - 평점과 댓글 기능을 사용하여 자신의 의견을 표현과 동시에 다른 사용자의 의견도 확인 가능
5) 음식점 등록 신청 기능 - 음식점 업주 입장에서 자신의 음식점을 등록신청하는 기능

Conception

Used skills

Frontend

● Html
● Css
● BootStrap

Backend

● Jsp
● Mysql
● Apache Tomcat

Result


Review:

Jsp와 Myspl를 이용하여 웹 사이트의 기본적인 기능들을 직접 개발할수 있었던 좋은 경험이자, Database에 관심을 가지게 된 계기가 된 프로젝트라고 생각합니다.
그러나 웹사이트 뿐만 아니라 사용자가 모바일을 이용하여 쉽게 접근 할 수 있도록 모바일 버전의 UI를 만들어 애플리케이션까지 개발했다면 훨씬 더 사용성과 접근성이 좋지 않았을까라는 생각에 들었습니다.
